Through-Going Muon (TGM) Tagger — check_tgm

Algorithm walkthrough of the WireCell-Prototype cosmic-rejection tagger.

What it does, in one sentence: Given the reconstructed 3-D cluster of an event and a candidate optical flash (which fixes the drift offset), check_tgm returns true if the cluster looks like a through-going muon — a cosmic track whose two ends both exit the detector's fiducial volume — rather than a contained neutrino interaction.

2. Inputs and key geometric concepts

The extreme points of the cluster

std::vector<std::vector<WCPoint>> out_vec_wcps = main_cluster->get_extreme_wcps();

out_vec_wcps is a list of endpoint groups. Each group out_vec_wcps.at(i) is a small bundle of candidate extreme points sitting at one "tip" of the cluster (the principal extreme first, then alternates). Group 0 and group 1 are the two ends of the cluster's main (longest) axis; further groups are other protrusions. The algorithm tests pairs of ends to see whether a straight muon trajectory connecting them would exit the detector at both ends.

The fiducial volume

inside_fiducial_volume(p, offset_x) (ToyFiducial.cxx:1743) returns whether a point is inside the active TPC, using a 2-D point-in-polygon test (pnpoly) in the x–y and x–z projections. The offset_x (derived from the flash time) shifts the cluster in x so the drift coordinate is physical. A point with both projections inside ⇒ inside; otherwise it is an exit point (touches anode, cathode, top/bottom, or upstream/downstream edge).

Wire-plane directions

TVector3 drift_dir(1,0,0);                                  // x = drift
TVector3 U_dir(0, cos60,  sin60);                           // U induction
TVector3 V_dir(0, cos60, -sin60);                           // V induction
TVector3 W_dir(0, 1, 0);                                    // W collection

These are used to detect track segments that run nearly parallel to a wire plane (a "prolonged" / isochronous direction), where reconstruction is unreliable and a true track end may be hidden — handled via signal-processing and dead-region checks.

3. High-level logic

The core idea: a through-going muon has two ends that both lie outside the fiducial volume, with a genuine track between them. The function searches over all pairs of endpoint groups (i, k) and tries to confirm that both ends exit, with safeguards against (a) gaps caused by dead channels / signal-processing holes, and (b) real neutrino topologies that merely poke a track out of the boundary.

length_limit = |end0 - end1|            // length of the main axis

for each pair of endpoint groups (i, k):
    determine if group i has a point OUTSIDE the FV  -> flag_p1_inside, p1_index
    determine if group k has a point OUTSIDE the FV  -> flag_p2_inside, p2_index

    CASE A: both ends are OUTSIDE (!p1_inside && !p2_inside)
        verify the connecting segment actually passes THROUGH the FV
        -> confirm it is not a neutrino candidate / handle flash type 2
        -> may return TRUE

    CASE B: at least one end is INSIDE
        only proceed if the pair is nearly the full main axis (angle > 75 deg, or i,k = 0,1)
        for each inside end, "push" it outward along its local Hough direction and
        re-test using dead-region + signal-processing checks
        if both ends then qualify as effectively outside -> may return TRUE

return false   // not a TGM

4. Step-by-step walkthrough

Step 0 — Setup lines 1333–1345

  • Grab the cluster's extreme-point groups.
  • Define drift & wire-plane unit vectors.
  • Compute length_limit = distance between the two principal extreme points (the main-axis length). Later, a candidate exit-to-exit chord must be at least 45% of this to count, rejecting short stubs.

Step 1 — Outer loops over endpoint-group pairs lines 1350–1374

For group i, scan its points; the first point found outside the FV sets flag_p1_inside = false and records p1_index. If every point is inside, the group is "inside". Same for group k (flag_p2_inside, p2_index). The pair (i,k) with k > i is then classified.

Step 2 — CASE A: both ends outside the FV lines 1392–1478

  1. Sanity-check the middle of the track. Sample 3 points at ¼, ½, ¾ along the segment from p1 to p2; flag_check is true if any interior sample is inside the FV. This confirms the two exit points are actually joined by a track that traverses the detector (not two unrelated edge hits).
  2. If the midpoint check passes (flag_check == true):
    • If main_flash->get_type()==2 (cathode-crossing flash, ambiguous t0): require the chord length > 0.45 × length_limit and that check_neutrino_candidate(...) returns false (i.e. the path does NOT look like a neutrino). For the principal pair (i,k)=(0,1) the stricter two-view check is enabled (flag_2view_check=true). → return true.
    • Otherwise (normal flash): return true directly — a clean through-going muon.
  3. If the midpoint check fails (track seems to leave the FV in the middle):
    • If there are only 2 endpoint groups total, accept anyway: return true.
    • Otherwise, re-test using the other endpoint groups as intermediate waypoints (flag_check_again): sample along p1 → other and other → p2. If no interior point is inside the FV (!flag_check_again) and the chord exceeds 45% of the main axis and it is not a neutrino candidate → return true.

Step 3 — CASE B: at least one end appears inside the FV lines 1479–1596

A reconstructed end can falsely look "inside" when the true exit is hidden by dead channels or a signal-processing gap. CASE B recovers these.

  1. Gate on geometry. Only proceed if the pair is essentially the full track: the connecting direction is within 15° of the cluster's principal PCA axis (|90° − angle(dir_test, dir_main)| > 75°) OR the pair is the principal pair (0,1). Otherwise this pair is skipped.
  2. Push each "inside" end outward. For an inside end p1:
    • Compute the local track direction via a Hough transform over a 30 cm neighbourhood (VHoughTrans), pointing outward.
    • Geometric consistency guard: skip the pair if this direction disagrees with the end-to-end direction (angle > 120° for p1; < 60° for p2).
    • Prolonged-track (isochronous) check: compute the angle of the direction to each wire plane (U, V, W). If the track is nearly parallel to a plane (angle1_1 < 10°, angle2_1 < 10°, or angle3_1 < 5°), require check_signal_processing(...) to pass — i.e. the apparent containment is not just a signal-processing hole.
    • Dead-region check: if the direction is far from the drift direction (angle > 60°), require check_dead_volume(...) — the extrapolated exit path is not dominated by dead channels.
    • If these checks fail, flag_p1_inside_p is cleared — the end is treated as effectively outside.
    The same procedure is applied to the other end p2 → flag_p2_inside_p.
  3. Decision. If after these corrections both ends are effectively outside (!flag_p1_inside_p && !flag_p2_inside_p):
    • flash type 2 → same neutrino-candidate + 45%-length guard as CASE A;
    • otherwise → return true.

Step 4 — Default line 1601

If no endpoint pair satisfied either case, the cluster is not a through-going muon: return false.

6. The flash-type = 2 (cathode-crossing) special case

When main_flash->get_type()==2 the flash↔cluster match is a cathode-crossing / ambiguous-t0 hypothesis, so the tagger is more conservative: it never returns true on geometry alone. It additionally demands that check_neutrino_candidate rejects the neutrino hypothesis and that the exit-to-exit chord is longer than 0.45 × length_limit. The principal pair (0,1) additionally enables the stricter U/V two-view path check. (get_type() is defined in data/inc/WCPData/Opflash.h:31.)

8. Summary & decision table

SituationExtra requirementResult

Both ends outside, midpoint fails, but no interior point inside via other tips not a neutrino candidate AND chord > 0.45 × main-axis TGM (true)

Take-away. check_tgm is fundamentally a both-ends-exit test made robust against detector imperfections: it (1) confirms a real track spans two FV exits, (2) rescues exits hidden by dead channels or signal-processing gaps using local Hough directions, and (3) refuses to tag anything that the neutrino-candidate path analysis flags as a plausible interaction — being especially strict for ambiguous (type-2) flashes.
